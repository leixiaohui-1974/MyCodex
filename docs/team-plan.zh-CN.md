# MyCodex Team Plan（优化版）

## 1. 目标

围绕 `Codex CLI + Agent Teams` 构建一个桌面协作系统，覆盖：

- 日常写作
- 科研流程
- 代码开发
- 项目管理与可视化结果预览
- 私有化协作（Gitea + 懒猫微服 + lzc-cli）

## 2. 核心路线

1. 本地桌面端作为统一入口（MyCodex）。
2. 团队仓库优先使用私有化 Gitea。
3. GitHub 作为可选外部冗余备份。
4. Agent Teams 作为默认执行引擎，所有任务可追踪到阶段与产物。

## 3. 为什么选 Gitea + 懒猫微服

- 私有化部署，数据自控。
- 内网穿透与开箱部署体验更适合团队内部快速协作。
- 成本可控，零月租可持续运行。
- 与 Git 工作流天然兼容，便于科研与开发的版本可追溯。

## 4. 团队协作机制（Git）

### 分支策略

- `main`：稳定主线
- `feat/<topic>`：功能分支
- `fix/<topic>`：缺陷修复
- `research/<topic>`：科研实验分支
- `docs/<topic>`：写作文档分支

### 提交规范

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `research: ...`
- `chore: ...`

### 合并规范

每次合并必须附带：

1. team plan 摘要
2. agent run summary
3. 风险与回滚说明
4. 验证日志或产物链接

## 5. 账号体系

### 当前策略（MVP）

- Gmail：继续作为主账号入口
- 微信：新增本地绑定入口，用于团队身份映射

### 设计原则

- 登录入口可并行（Google / 微信）
- 本地端保存账号映射元数据
- 远端协作通过仓库权限与项目映射完成，不强耦合单一 OAuth 提供方

## 6. 执行阶段

1. 需求与范围
2. 方案设计
3. 实施
4. 测试验证
5. 审查加固
6. 发布前检查

每个阶段都由 Agent Teams 自动生成产物并可视化。

## 7. 验收标准

- 可从桌面端一键执行 `team plan/run/resume/status`
- 可在桌面端发起 LZC/Gitea 操作
- 可初始化团队 Git 协作规范
- 可绑定 Gitea 远程并推送
- 可记录 Gmail / 微信本地身份映射
- 可在 UI 中查看执行日志与产物预览
